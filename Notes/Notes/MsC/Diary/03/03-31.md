How cumulative histogram solution works:
Starting off with histogram, (C, nbins)
forms cumulative sum, meaning that each bin corresponding to a pixel value get counts of pixels from past columns, resulting in the last bin showing the total amount of pixels.
Then divide this cumsum with the total amount of pixels, so the last column is 1 in probability.

![[Pasted image 20260331120201.png]]

gets this shape. Then thresholding on bins where probability is over percentile
`P(X > x) > percentile`

CDF plots tells what fraction of counts are of this value, so in above plot, around 50% of values are below value 0.

So to find 90th percentile, pick 90% on y-axis, and find the x-value, and that is the value threshold for 90th percentile

```
    idx = np.argmax(cdf_ok >= (percentile / 100.0), axis=1)

    bin_width = (vmax[ok] - vmin[ok]) / global_hist.shape[1]
    out[ok] = (vmin[ok] + idx * bin_width).astype(np.float32)
```
cdf_ok >= percentile returns a boolean, and np.argmax looks at the array as 0, and 1s, and returns the first max in the array, meaning that the bins look like
`[0,0,1,1,1,1]`
So picks the first bin
The calculation extracts actual pixel value from index, so if its bin 5, and lowest value is say 1, then its 5 widths added to the lowest value of first bin, resulting in the value of the bin of that index.
