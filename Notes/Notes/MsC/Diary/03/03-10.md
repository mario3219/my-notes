It was easy to implement KRONOS into i2c, but I need to read about the preprocessing requisites.

KRONOS requires marker metadata, with mean and std. It has its own pretrained metadata, and is validated against users metadata. The metadata has to be generated by yourself, and is okay to generate from training data as a preprocessing step.

The pretrained metadata is divided by their ids, 1 to 512. Nuclear markers have ids 1 to 127, and non-nuclear 128-512. The pretrained metadata occupies ids in this range, but only 175 are given on even ids. The non-even ids are left for users to input, if their markers are missing. If there are markers missing in pretrained not present in user metadata, the user is allowed to create their own metadata for the marker, and assign it to an odd id in close grouping to another biologically similar marker.

For example, CD20 is not present, but CD8 is. CD8 has marker id 60. The user makes their own metadata for that marker, and is allowed to assign it id 61, because they are biologically similar.

**IMPORTANT**
When making your own metadata, make sure intensities are in range of 0-1 in each image before computing mean and std.
