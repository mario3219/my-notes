"_Text encoder_: For the text encoder, we used a dense model distilled from the open-source DeepSeek-R1 ([55](https://pmc.ncbi.nlm.nih.gov/articles/PMC11601246/#R55)) model built on the Llama-3.3–70B-Instruct model as the LLM expert explainer and its corresponding embedder as the LLM embedder. We prompt the explainer to provide detailed descriptions of each marker type including its general description, alternative names, and cell type correspondence. The resulting 8192-dimensional embeddings are linearly mapped to 256 dimensions to match the image embeddings"

```
import torch
import torch.nn.functional as F
from transformers import AutoTokenizer, AutoModelForCausalLM

model_name = "deepseek-ai/DeepSeek-R1-Distill-Llama-70B"   # huge, example only
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.float16,
    device_map="auto"
)

def deepseek_pseudo_embed(texts, max_length=512):
    batch = tokenizer(
        texts,
        padding=True,
        truncation=True,
        max_length=max_length,
        return_tensors="pt"
    ).to(model.device)

    with torch.no_grad():
        out = model(**batch, output_hidden_states=True)

    hidden = out.hidden_states[-1]  # [B, T, H]
    mask = batch["attention_mask"].unsqueeze(-1)
    pooled = (hidden * mask).sum(dim=1) / mask.sum(dim=1).clamp(min=1)
    pooled = F.normalize(pooled, dim=1)
    return pooled
```

Adding my own channel embeddings

* Marker embeddings gets loaded from:
	`marker_embeddings-deepseek-r1-70b-llama-distill-q4_K_M.json`
* A zeros array gets created with similar shape
* The loaded json is a dictionary, and gets iterated for key and value
* The key gets checked in property `dct_config.marker2idx`
* The property is initialized as a dictionary with key ch and value idx from variable `marker2embedding`
* The variable is initialized with method `get_channel_embedding` BUT using a default model, this time `"deepseek-r1-70b-llama-distill-q4_K_M"` (the same json file)

In conclusion, it loads embeddings formed from a desired model. If the embeddings are not present in the default embeddings to index, it is skipped.

If there are 75 channels, each of these gets their index, and the embeddings are stored for each. These get passed to the model.

The model uses these embeddings to form markernameembeddinglayer, and initializes these as tensors in embed_layer.

When its time to pass images through, a channel_names array is needed, which is index sensitive to what channel is what.

`get_channel_masking` gets called, the channel_names gets checked in `channel_mapping`, which is just a list of supported names. If it exists, `channel_masking` gets appended with `False`, and `channel_names_standard` gets appended with the channel name standard name (for example if it is named cyto13 in channel_names, it gets the standard name Cytokeratin13 as described in channel_mapping through Cyto13 : Cytokeratin13)

The channel_names_standard gets passed to function get_ch_idx, together with marker2idx property of dct_config (embeddings for each channel)

A new array is formed, where it extracts the embeddings for each channel name. So if channel names is (cyto13, cd3, cd8, panck) it gets embeddings and stores it in corresponding index, and pads the array to equal size as max channels with -1 for the rest. This array is later used when forming image embeddings.
